 1. Создать внешний репозиторий c названием TXT 
------------------------------------
------------------------------------
 2. Клонировать репозиторий TXT на локальный компьютер - git clone https://github.com/VladimirKolosov/TXT.git
------------------------------------
------------------------------------
 3. Внутри локального TXT создать файл “new.txt” - nano new.txt
------------------------------------
------------------------------------
 4. Добавить файл под гит - git add new.txt
------------------------------------
------------------------------------
 5. Закоммитить файл - git commit -m"new file"
------------------------------------
------------------------------------
 6. Отправить файл на внешний GitHub репозиторий - git push
------------------------------------
------------------------------------
 7. Отредактировать содержание файла “new.txt” - nano new.txt
------------------------------------
------------------------------------
 8. Отправить изменения на внешний репозиторий - git commit -am"changed file", git push
------------------------------------
------------------------------------
 9. Создать файл preferences.txt - nano preferences.txt
------------------------------------
------------------------------------
 10. В файл preferences.txt” добавить информацию о своих предпочтениях - открыть ранее созданный файл и внести в него инфу
------------------------------------
------------------------------------
 11. Создать файл sklls.txt добавить информацию о скиллах которые будут изучены - nano skills.txt
------------------------------------
------------------------------------
 12. Сделать коммит в одну строку - git add., git commit -m"new files"
------------------------------------
------------------------------------
 13. Отправить сразу 2 файла на внешний репозиторий - git push
------------------------------------
------------------------------------
 14. На веб интерфейсе создать файл bug_report.txt - в репозитории нажать Add File и потом выбрать create new file, потом нажать commit new file
------------------------------------
------------------------------------
 15. Сделать Commit changes (сохранить) изменения на веб интерфейсе - открыть ранее созданый файл внести в него информацию и нажать commit changes
------------------------------------
------------------------------------
 16. На веб интерфейсе модифицировать файл bug_report.txt, добавить баг репорт в формате TXT
------------------------------------
------------------------------------
 17. Сделать Commit changes (сохранить) изменения на веб интерфейсе - после внесенной информации в 16 пунтке, нажать на кнопку сommit changes.
------------------------------------
------------------------------------
 18. Синхронизировать внешний и локальный репозиторий TXT - git pull
